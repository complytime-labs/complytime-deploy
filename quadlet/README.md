# Podman Quadlet Deployment

Run ComplyTime locally using rootless Podman with systemctl --user. No OpenShift, no CRC, no cluster — just Podman on Linux.

## Prerequisites

| Tool | Version | Purpose |
|------|---------|---------|
| `podman` | >= 4.4 | Container runtime with Quadlet support |
| `systemctl --user` | any | Manages container lifecycle |
| `openssl` | any | Generates TLS certificates (skip with `--no-tls`) |

**Enable user lingering** (allows services to run after logout):
```bash
loginctl enable-linger $(whoami)
```

## Quick Start

```bash
# Generate certs, configs, and install quadlet units
task quadlet:setup

# Or use a local collector image instead of the upstream default
COLLECTOR_IMAGE=localhost/complybeacon/collector:latest task quadlet:setup

# Start all services
task quadlet:start

# Check status
task quadlet:status

# Access services
#   Grafana:   https://localhost:3000 (accept self-signed cert warning)
#   Collector: https://localhost:4318
#   RustFS:    http://localhost:9001 (rustfsadmin/rustfsadmin)
```

## TLS Modes

### TLS Enabled (Default)

The setup script generates a self-signed CA and per-service certificates. Services communicate over HTTPS with mutual TLS, matching the production OpenShift deployment pattern.

```bash
task quadlet:setup
```

### TLS Disabled (Debugging)

All services communicate over plaintext HTTP. Useful for debugging with tools like `curl` without certificate hassle.

```bash
task quadlet:setup -- --no-tls
```

To switch modes, tear down first:
```bash
task quadlet:teardown
task quadlet:setup           # or: task quadlet:setup -- --no-tls
task quadlet:start
```

## Service Architecture

```
  localhost:4318 (OTLP HTTP)
  localhost:8088 (Webhook)
       |
       v
  complytime-collector
       |
       v
  complytime-loki <---------- complytime-grafana
                                localhost:3000
       |
       v
  complytime-rustfs (S3)
  localhost:9000/9001
```

All containers run on a shared `complytime` Podman network with DNS resolution by container name. RustFS provides S3-compatible object storage for evidence export.

## Managing Services

```bash
# Start/stop
task quadlet:start
task quadlet:stop

# Status of all services
task quadlet:status

# Logs (via journalctl)
task quadlet:logs -- collector
task quadlet:logs -- loki
task quadlet:logs -- grafana
task quadlet:logs -- rustfs

# Or use systemctl directly
systemctl --user status complytime-collector
systemctl --user restart complytime-loki
journalctl --user -u complytime-grafana -f
```

## File Layout

```
quadlet/
├── templates/                   # Quadlet unit templates (checked in)
│   ├── complytime.network               # Shared container network
│   ├── complytime-loki-storage.volume   # Persistent Loki data
│   ├── complytime-grafana-storage.volume # Persistent Grafana data
│   ├── complytime-rustfs-storage.volume # Persistent RustFS data
│   ├── complytime-loki.container
│   ├── complytime-collector.container
│   ├── complytime-grafana.container
│   └── complytime-rustfs.container      # RustFS S3-compatible storage
├── configs/                     # Quadlet-specific config templates
│   ├── grafana-datasource-tls.yaml     # With @@CA_CERT@@ placeholder
│   └── grafana-datasource-notls.yaml   # Plain HTTP variant
└── runtime/                     # Generated by setup (gitignored)
    ├── certs/                   # Self-signed CA and per-service certs
    │   ├── ca.crt, ca.key
    │   ├── loki/tls.crt, tls.key, ca.crt
    │   ├── collector/...
    │   └── grafana/...
    ├── configs/                 # Final configs (TLS or no-TLS variant)
    └── env/                     # Environment files for containers
```

The setup script reads service configs from `base/` and `overlays/local/configs/` (shared with Kustomize) and generates quadlet-specific variants at runtime.

## Cleanup

```bash
# Full teardown — stops services, removes units, volumes, certs, configs
task quadlet:teardown
```

This removes:
- All systemd unit files from `~/.config/containers/systemd/`
- Podman volumes (`complytime-loki-storage`, `complytime-grafana-storage`, `complytime-rustfs-storage`)
- The `complytime` Podman network
- Everything in `quadlet/runtime/`

## Differences from OpenShift Deployment

| Feature | OpenShift (Kustomize) | Podman Quadlet |
|---------|----------------------|----------------|
| TLS | OpenShift Service CA (automatic) | Self-signed CA (generated by setup script) |
| Auth | OIDC + JWT (stage/prod), disabled (local) | Always disabled |
| Networking | Kubernetes Services + Routes | Podman network + localhost ports |
| Storage | PersistentVolumeClaims | Podman named volumes |
| S3 evidence | RustFS (local) or AWS S3 (prod) | RustFS (always) |
| Lifecycle | `skaffold dev` / `task sk:dev` | `systemctl --user` / `task quadlet:start` |
| Secrets | Kubernetes Secrets | Environment files |
| DNS | `service-name:port` | `complytime-service:port` |

## Troubleshooting

### Services Won't Start

```bash
# Check for errors in the generated unit
systemctl --user status complytime-collector

# View full logs
journalctl --user -u complytime-collector --no-pager

# Verify quadlet files are installed
ls ~/.config/containers/systemd/
```

### Image Pull Failures

```bash
# Check if images are accessible
podman pull quay.io/huiwang/collector:latest

# Use a local image if the upstream isn't available
COLLECTOR_IMAGE=localhost/complybeacon/collector:latest task quadlet:setup
task quadlet:restart

# For private registries, login first
podman login quay.io
```

### Port Conflicts

If ports 3000, 4318, or 8088 are already in use:
```bash
# Find what's using the port
ss -tlnp | grep 3000

# Stop the conflicting service, then restart
task quadlet:start
```

### Cert Issues (TLS Mode)

```bash
# Regenerate certs
task quadlet:teardown
task quadlet:setup
task quadlet:start

# Or switch to no-TLS for debugging
task quadlet:teardown
task quadlet:setup -- --no-tls
task quadlet:start
```
